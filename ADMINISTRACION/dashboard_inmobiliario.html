1. Resumen Ejecutivo

Este documento presenta un analisis completo de la operacion actual de administracion de propiedades y una propuesta detallada para desarrollar un dashboard administrativo inmobiliario que transforme la gestion manual basada en Excel en un sistema digital centralizado, moderno y escalable.

Oportunidad Clave

La operacion actual gestiona 19 inmuebles (apartamentos, casas, habitaciones y locales) mediante hojas de calculo Excel. Este modelo, funcional en etapas iniciales, presenta limitaciones criticas en escalabilidad, visibilidad de datos y automatizacion que afectan directamente la rentabilidad y el crecimiento del negocio.

La investigacion abarco mas de 15 plataformas de referencia global (AppFolio, Buildium, TenantCloud, Yardi, Rentec Direct, OhMyRent, BReal, entre otras) y analisis de KPIs estandar de la industria inmobiliaria. A partir de este analisis, se define una propuesta funcional especifica y un plan de implementacion por fases de 16 semanas.

Objetivo del documento: Servir como especificacion funcional completa para que Antigravity desarrolle el dashboard administrativo inmobiliario, eliminando ambiguedades y acelerando el proceso de desarrollo.

2.3 Procesos Identificados y Sus Fricciones

A partir del analisis del Excel se identificaron los siguientes procesos operativos y sus puntos de friccion:

Procesos y fricciones identificadas
Proceso Como se hace hoy Friccion principal

Control de ocupacion Etiquetas manuales "DESOCUPADO", "PREAVISO", "CONTRATO NUEVO" Sin indicador visual, sin metrica de vacancia, sin forecast

2.4 Limitaciones del Sistema Actual

Limitaciones Criticas Identificadas
  • No hay visibilidad en tiempo real: Los datos se actualizan manualmente, generando retrasos y versiones desactualizadas.
  • Sin alertas ni notificaciones: Los aumentos de canon, vencimientos de contrato y pagos pendientes dependen 100% de la memoria del equipo.
  • Sin metricas ni KPIs: No es posible calcular rapidamente tasa de ocupacion, rentabilidad neta, morosidad o tendencias de ingresos.
  • Escalabilidad limitada: Cada nuevo inmueble agrega complejidad exponencial a las hojas de calculo.
  • Riesgo de datos: Sin respaldo automatico, sin control de acceso, sin historial de cambios.
  • Sin portales para propietarios ni arrendatarios: Toda comunicacion es manual (WhatsApp, llamadas).
  • Conciliacion bancaria manual: Cada transaccion se registra a mano, sin importacion bancaria.

3.3 KPIs Clave para Administracion de Propiedades

Segun la investigacion de la industria (IREM, NAA, NMHC), estos son los KPIs que todo dashboard inmobiliario debe monitorear:

KPIs fundamentales para dashboards inmobiliarios
KPI Formula Por que importa Frecuencia
Tasa de Ocupacion (Unidades ocupadas / Unidades totales) x 100 Indicador primario de salud financiera. < 90% requiere accion inmediata Diaria/Semanal
Tasa de Recaudacion (Canon recaudado / Canon debido) x 100 Mide eficiencia de cobro. Ideal >95%. Afecta flujo de caja directamente Mensual
Ingreso Neto de Operacion (NOI) Ingresos totales - Gastos operativos Margen de rentabilidad real. Base para valuacion de propiedades Mensual/Trimestral
Tasa de Renovacion (Renovaciones / Contratos por vencer) x 100 Costo de rotacion: $2,500-4,000 por unidad. Alta retencion = mayor profit Mensual
Dias Promedio para Arrendar Dias totales vacantes / Unidades arrendadas Mide eficiencia de marketing y precios. Menor = mejor Mensual
Tasa de Morosidad (Canon en mora / Canon total) x 100 Alerta temprana de problemas de cobro o calidad de inquilinos Semanal
Costo de Mantenimiento por Unidad Gastos mantenimiento / Unidades totales Identifica propiedades con problemas recurrentes de infraestructura Mensual
Crecimiento de Canon Efectivo Variacion canon promedio mes actual vs mismo mes ano anterior Mide si los aumentos estan alineados con inflacion y mercado Trimestral

3.4 Mejores Practicas de Diseno UI/UX

El analisis de templates y UI kits de property management revelo patrones consistentes en los mejores dashboards:

Patrones de diseno identificados:
  • KPI Cards superiores: Fila de 3-4 cards con metricas principales (ocupacion, recaudacion, NOI, propiedades activas)
  • Graficos temporales: Barras o lineas mostrando tendencias de ingresos, gastos y ocupacion
  • Lista de transacciones recientes: Tabla con las ultimas transacciones, pagos y alertas
  • Calendario de eventos: Contratos por vencer, aumentos de canon, pagos pendientes
  • Estados visuales: Semaforos para ocupacion (verde/amarillo/rojo), badges para estados de contrato
  • Filtros por propiedad/tiempo: Selectores para cambiar vista entre inmuebles, periodos y categorias
  • Sidebar navigation: Menu lateral con Dashboard, Propiedades, Arrendatarios, Finanzas, Reportes, Configuracion

4. Propuesta de Dashboard

4.1 Vision General del Sistema

El dashboard propuesto es un sistema web de administracion de propiedades disenado especificamente para la operacion identificada. Reemplaza el Excel con una plataforma centralizada que automatiza procesos, genera alertas inteligentes y proporciona visibilidad total del portafolio en tiempo real.

Principios de Diseno
  • Simplicidad operativa: Cada pantalla debe resolver una tarea especifica sin sobrecarga cognitiva
  • Visibilidad inmediata: Los KPIs criticos deben estar siempre visibles en menos de 3 segundos
  • Accionable: Cada alerta debe tener una accion clara asociada
  • Escalable: El sistema debe funcionar igual de bien con 19 o 100 propiedades
  • Mobile-first: Acceso desde cualquier dispositivo, especialmente celular

4.3 Especificacion de Widgets y Componentes

4.3.1 Dashboard Principal - Vista Detallada

El dashboard principal es la pantalla de inicio y debe mostrar la siguiente informacion organizada en secciones:

Fila 1: KPI Cards (4 cards)

  • Total Inmuebles: Numero con variacion vs mes anterior (ej: "19 / 0%")
  • Tasa de Ocupacion: Porcentaje con barra de progreso visual (ej: "89% / 17 ocupados de 19")
  • Recaudacion Mes: Valor en COP con meta vs real (ej: "$9.8M recaudado / $11M meta")
  • NOI Mes: Ingreso neto con tendencia (ej: "$6.2M / +3.2% vs mes anterior")

Fila 2: Graficos (2 columnas)

  • Grafico de Ingresos vs Gastos (ultimos 12 meses): Barras agrupadas mostrando ingresos por canon vs gastos operativos + gastos oficina
  • Grafico de Ocupacion (ultimos 12 meses): Linea mostrando % de ocupacion mes a mes con meta del 95%

Fila 3: Alertas y Actividad (2 columnas)

  • Panel de Alertas: Lista de alertas prioritarias ordenadas por urgencia:
    • Contratos por vencer en los proximos 60 dias
    • Canones pendientes de pago (morosidad)
    • Aumentos de canon programados para este mes
    • Depositos pendientes de devolucion
  • Actividad Reciente: Ultimas 10 transacciones registradas con fecha, descripcion, monto e inmueble asociado

Fila 4: Calendario y Mapa (2 columnas)

  • Calendario de Eventos: Vista mensual con indicadores de colores para: inicios de contrato, vencimientos, aumentos de canon, pagos programados
  • Resumen por Propiedad: Tabla compacta con cada inmueble, estado (ocupado/desocupado/preaviso), canon actual y proximo evento

4.3.2 Pantalla de Detalle por Inmueble

Al seleccionar un inmueble del catalogo, se muestra:

  • Info basica: direccion, tipo, propietario, administrador asignado
  • Estado actual: ocupado/desocupado, arrendatario actual, canon mensual
  • Contrato activo: fecha inicio, duracion, fecha vencimiento, deposito, dias restantes
  • Historial financiero: grafico de ingresos/gastos de los ultimos 12 meses
  • Historial de arrendatarios: lista de inquilinos anteriores con fechas
  • Documentos: fotos del inmueble, contratos PDF, inventarios, certificados
  • Mantenimiento: tickets historicos y activos con costos

4.3.3 Pantalla de Finanzas

  • Registro de transacciones: Tabla filtrable con fecha, descripcion, categoria, inmueble, tipo (ingreso/gasto), monto
  • Categorias: Casa Azul, Habitacion Azul, Apto Azul, Edificio, Otros (igual que el Excel actual) + subcategorias
  • Grafico de gastos por categoria: Torta o barras horizontales
  • Comparativa mes a mes: Tabla de doble entrada (meses x categorias)
  • Exportacion: PDF y Excel para contador/propietarios

6. Conclusiones y Recomendaciones

La transformacion del sistema actual basado en Excel hacia un dashboard administrativo inmobiliario representa una oportunidad de crecimiento significativa para la operacion. Los beneficios esperados incluyen:

Beneficios esperados del nuevo sistema
Area Beneficio Impacto Estimado
ProductividadEliminacion de tareas manuales de registro y calculo-70% tiempo en tareas administrativas
CobroAlertas tempranas de morosidad y recordatorios automaticos+15% tasa de recaudacion puntual
VacanciaAlertas de vencimiento y preaviso con 60 dias de anticipacion-30% dias promedio de vacancia
DecisionesKPIs en tiempo real para ajustes de precio y estrategiaMayor rentabilidad neta del portafolio
PropietariosReportes automaticos y portal de transparencia+40% satisfaccion del cliente
EscalabilidadCapacidad de crecer sin aumentar proporcionalmente el equipoDe 19 a 50+ inmuebles sin friccion
Recomendacion Prioritaria

Iniciar inmediatamente con la Fase 1 (Fundacion y Datos). La migracion de datos desde Excel es el cuello de botella critico: mientras mas tiempo pase sin migrar, mas desactualizados quedaran los datos y mayor sera el esfuerzo de sincronizacion. Una vez los datos esten en la base de datos, el desarrollo de las interfaces es iterativo y puede mostrar progreso visible semana a semana.

El equipo de desarrollo debe priorizar la exactitud de los datos financieros sobre la estetica. Un dashboard con KPIs correctos y alertas puntuales genera mas valor operativo que uno visualmente sofisticado pero con datos inconsistentes. La recomendacion es validar cada interfaz con datos reales antes de avanzar al siguiente modulo.

Proximos pasos sugeridos:
  1. Semana 0: Kickoff con Antigravity, acceso al Excel completo, definicion de stack tecnologico
  2. Semana 1: Inicio Fase 1 - Modelo de datos y migracion
  3. Revision semanal: Demos cada viernes con el equipo administrativo para validar avances
  4. Go-live parcial: Al finalizar Fase 2, usar el sistema en paralelo con Excel por 2 semanas
  5. Go-live total: Al finalizar Fase 4, Excel como respaldo unicamente
